## 2. 产品背景与目标

### 2.1 产品背景

平台面向高校职业能力服务场景，当前已实现统一认证、用户与角色权限、个人资料、操作审计、API Key、API 调用统计和 Swagger API 文档。系统管理员是功能最完整的用户，学生、教师、数据分析员和学院管理员使用同一套前端框架，但菜单由角色与权限动态决定。

### 2.2 本次设计目标

1. 将当前基础管理台升级为专业、安静、可信、适合长期高频操作的企业级界面。
2. 提高表格扫描、筛选、创建、编辑、授权、审计和危险操作的效率。
3. 在不增加后端能力的前提下，清晰呈现管理员与普通用户的差异。
4. 让桌面、平板和手机页面都不溢出、不重叠，核心操作始终可达。
5. 输出的页面结构必须便于后续转换为 Vue 3 + Element Plus 组件。

### 2.3 不属于本次范围

- 不新增就业趋势、岗位推荐、课程、简历、消息、通知、全局搜索或数据大屏。
- 不虚构当前接口没有返回的折线图、饼图、排名或同比数据。
- 不改变后台 API、字段、角色编码、权限编码或认证方式。
- 不生成营销落地页，不生成巨型 Hero，不生成深色主题。
- 不使用玻璃拟态、霓虹、渐变背景、装饰光球、3D、插画式仪表盘或大面积阴影。

## 5. Stitch 输出要求

### 5.1 画板和响应式输出

1. 所有页面先生成 1440 x 1024 桌面画板。
2. 所有页面均定义 390 x 844 手机布局，不允许只缩小桌面画板。
3. 至少额外生成一个 900px 宽的侧边栏抽屉状态，用于验证平板布局。
4. 表格管理页至少展示 8 行真实感示例数据、加载状态和空状态。
5. 弹窗、抽屉、确认框必须作为对应页面的状态变体生成。
6. 如果一次生成数量受限，严格按以下批次在同一 Stitch 项目中继续生成，不得省略后续批次：
   - A：设计系统、登录、注册。
   - B：全局应用框架、管理员工作台、普通用户工作台、个人资料。
   - C：用户管理、角色权限、操作日志及其弹窗/抽屉。
   - D：API Key、API 调用统计、API 文档及其弹窗。
   - E：全部页面的手机变体和移动菜单。

### 5.2 输出可移植性

- 使用标准 Flexbox/Grid 布局，不使用 Canvas、WebGL 或只能在特定框架运行的布局技巧。
- 图标使用一致的 1.75px 至 2px 描边线性图标；不要手绘 SVG。后续实现会映射到 `@element-plus/icons-vue`。
- 所有页面和组件必须使用本文件定义的颜色、间距、字号和圆角 token。
- 若 Stitch 输出代码，组件名采用本文件中的英文画板/组件名，界面文案保持简体中文。
- 登录页图片必须作为可替换的独立位图资源输出或清楚标记资源引用，不要把图片编码成 CSS 或 SVG。
- 不依赖新接口。示例数据只用于设计展示。

## 6. 视觉设计系统

### 6.1 视觉气质

关键词：可信、克制、清晰、精确、数据导向、教育行业、长期使用。

- 主体是浅色工作界面，不是深色开发者工具。
- 信息层级主要依赖留白、细边框、文字权重和少量表面色差。
- 墨绿只用于品牌、主操作、选中项和焦点，不铺满页面。
- 蓝色用于中性信息，黄色用于警告，红色用于失败和危险操作。
- 页面区块采用全宽或无外框布局；卡片仅用于独立重复指标、弹窗和确实需要边界的工具。
- 不允许卡片嵌套卡片，不允许每个小区块都漂浮成卡片。

字体栈：`Inter, "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`。

| 用途 | 字号/行高 | 字重 |

| 页面 H1 | 24/32px | 650 |
| 分区 H2 | 18/26px | 600 |

### 6.4 间距、尺寸与圆角

- 4px 基础网格：`4, 8, 12, 16, 20, 24, 32, 40`。
- 桌面内容区外边距 24px；宽屏可增加到 32px。
- 手机内容区外边距 16px。
- 顶部栏 64px；手机顶部栏 56px。
- 展开侧栏 240px；折叠侧栏 64px；手机抽屉 280px。
- 默认输入框和按钮高度 40px；紧凑按钮 32px；手机主操作至少 44px。
- 表格表头 44px，数据行 48px。
- 控件圆角 6px，卡片/弹窗最多 8px，小标签 4px。
- 状态标签可使用 999px 胶囊圆角；普通按钮禁止胶囊形。
- 阴影只用于下拉菜单和弹窗：`0 12px 32px rgba(18, 38, 31, 0.12)`。页面卡片不用重阴影。

## 7. 全局应用框架 AppShell

### 7.1 桌面侧边栏

- 顶部品牌区：36px 的 `CA` 字母标识、`职业能力平台`、副标题 `管理控制台`。
- 导航分为三个视觉组，组标题较弱，并在组内没有可见菜单时整体隐藏：
  - 概览：工作台、个人资料。
  - 系统管理：用户管理、角色权限、操作日志。
  - 开放平台：API Key、调用统计、API 文档。
- 当前页面使用 `brand-50` 背景、`brand-600` 图标与左侧 3px 指示线。
- 折叠时只保留标识和图标，所有图标显示 Tooltip。
- 侧栏折叠按钮位于顶部栏，不在侧栏底部重复放置。

### 7.2 顶部栏

- 左侧：侧栏切换图标、弱化的分组名/面包屑、当前页标题。
- 右侧：圆形用户头像、显示名、下拉箭头。
- 用户菜单仅包含 `个人资料`、分隔线、`退出登录`。
- 不生成通知铃铛、全局搜索、帮助中心或语言切换，因为当前没有对应功能。

### 7.3 主内容区

- 页面头部无卡片：左侧页面标题和可选说明，右侧页面级操作。
- 页面头部下方是实际业务表面。表格页只使用一个表格容器，不再套一层装饰卡片。
- 最大内容宽度不强制限制；表格页使用可用宽度。工作台和个人资料在超宽屏下最大 1320px。
- 页面切换不做夸张动画，仅允许 120ms 至 180ms 的淡入或背景色过渡。

### 7.4 手机和平板

- 小于 900px 时隐藏固定侧栏，顶部左侧显示菜单图标。
- 菜单从左侧抽屉打开，点击导航项后自动关闭。
- 手机顶部仅显示页面标题；用户名隐藏但头像保留。
- 页面操作较多时，保留一个主按钮，次要操作进入 `更多` 菜单；功能不能丢失。

## 8. 通用组件规范

### 8.1 按钮和图标

- 主按钮：墨绿实底、白字，用于每个页面唯一的主要提交或创建动作。
- 次按钮：白底、细边框、深色文字。
- 文本/图标按钮：用于表格行操作；图标按钮固定 32 x 32px，不因内容变化改变布局。
- 删除操作使用红色图标/文字，并在执行前确认。
- 熟悉的动作只使用图标：折叠菜单、查看、编辑、重置密码、删除、复制、刷新；所有图标按钮必须有 Tooltip 和无障碍名称。
- 创建、保存、查询、导入等明确命令使用图标加文字。

### 8.2 表单

- 标签放在输入框上方，必填项有清晰星号。
- 焦点显示 2px `focus` 外环，不能只改变边框色。
- 错误信息位于字段下方，红色文字配错误图标，不能导致相邻控件重叠。
- 密码输入保留显示/隐藏按钮。
- 二元状态使用 Switch；角色类型使用 Segmented Control；角色多选使用多选 Select。
- 数字限流使用 Stepper/Input Number；过期时间使用日期时间选择器。

### 8.3 筛选栏 FilterToolbar

- 桌面同一行：输入/选择控件在左，`查询` 和 `重置` 紧随其后。
- 查询按钮是次要按钮，不与页面级创建主按钮争夺视觉层级。
- 手机垂直排列，输入占满宽度，查询与重置组成双列按钮行。
- 重置后回到第 1 页。

### 8.4 数据表 DataTable

- 表头使用 `surface-muted`，文字 12px/600；正文 13px。
- 行之间使用极细分隔线，可保留非常轻的斑马纹，不使用大色块。
- Hover 行使用 `brand-50`，选中不改变行高。
- 长文本截断并通过 Tooltip 查看完整内容。
- 桌面右侧操作列固定；手机表格允许水平滚动，行操作合并到三个点菜单。
- 空状态在表格内部显示简洁图标、说明和可选主动作，不使用大型插画。
- 加载时保留表头，数据区显示 6 行 Skeleton，避免布局跳动。

### 8.5 分页

- 桌面右对齐：总数、每页数量、上一页、页码、下一页。
- 页大小选项固定为 `10 / 20 / 50 / 100`。
- 手机简化为总数、上一页、当前页、下一页；不得横向溢出。

### 8.6 弹窗、抽屉和反馈

- 创建/编辑使用 500px 至 640px Dialog。
- 查看详情和权限树使用右侧 Drawer，桌面宽 480px 至 520px，手机占满宽度。
- Dialog 底部操作固定右对齐：取消在前，主操作在后。
- Drawer 内容滚动时，标题栏和底部操作栏保持可见。
- 成功使用短 Toast；接口错误使用错误 Toast；危险操作使用 Confirm Dialog。
- 提交期间按钮显示加载并禁用重复提交。

## 9. 页面详细规格

### 9.1 登录页 `Auth / Login`

**目标**：让管理员、学生、教师和其他已分配账号的用户从同一入口登录。

**桌面布局**：

- 全屏双区域，左侧约 46%，右侧约 54%；右侧表单内容最大宽 380px。
- 左侧使用一张清晰、可替换的真实位图：高校学生与教师在职业发展中心协作查看数据，不模糊、不做纯氛围图。图片应能看清真实场景。
- 图片上只使用轻量纯色遮罩，叠加 `职业能力大数据服务平台`、标题 `就业数据与能力管理` 和短句 `高校就业分析、岗位能力洞察和平台权限管理`。文字不放在卡片里。
- 右侧是白色表单面，不使用漂浮卡片。标题建议为 `登录职业能力平台`，副标题 `使用你的平台账号继续`。

**表单和交互**：

- 用户名，前置用户图标，自动完成 `username`。
- 密码，前置锁图标，显示/隐藏按钮，自动完成 `current-password`。
- 主按钮 `登录`，宽度 100%，支持 Enter 提交和加载状态。
- 底部链接：`还没有账号？创建账号`，前往 `/register`。
- 不展示默认管理员账号密码。
- 错误状态必须覆盖：必填校验、用户名或密码错误、账号临时锁定 15 分钟、无法连接认证服务、未知失败。
- 登录成功后保持现有 redirect 参数行为。

**手机布局**：

- 真实场景图片变为顶部 150px 高品牌带，表单紧随其后，首屏内可见登录按钮。
- 不能只保留图片而把表单挤出首屏。

### 9.2 注册页 `Auth / Register`

**目标**：只允许创建学生或教师账号。

**布局**：

- 居中的单一表单工具面，桌面最大宽 760px，白色背景、8px 圆角、细边框。
- 头部左侧为平台名和 `创建平台账号`，右侧为带左箭头的 `返回登录`。
- 桌面表单两列；手机单列全宽。

**字段**：

1. 账号类型：Segmented Control，`学生` / `教师`，默认学生。
2. 用户名：4 至 20 位字母、数字或下划线。
3. 密码：6 至 20 位，带显示/隐藏。
4. 确认密码：必须与密码一致。
5. 姓名。
6. 邮箱：邮箱格式校验。
7. 所属学院：桌面跨两列。
8. 主按钮：`创建账号`，全宽，加载时禁用。

注册成功后显示成功 Toast 并返回 `/login`。不要增加管理员、分析员或学院管理员自助注册选项。

### 9.3 工作台 `Console / Home`

**共同布局**：

- 页面标题 `欢迎，{displayName}`，标题下用次要文字显示所属学院；未设置时显示 `未设置所属学院`。
- 第一行是 3 个同级摘要项，不再套外层卡片：`当前角色`、`有效权限`、`账号状态`。
- 账号状态使用绿色圆点和 `正常`，停用使用红色圆点和 `停用`。
- 下方是 `快捷入口`，每项为稳定 48px 高的命令行式入口：左图标、文字、右箭头。
- 不增加任何虚构业务图表。

**管理员状态**：

- 示例值：当前角色 `1 项`、有效权限 `10 项`、账号状态 `正常`。
- 快捷入口：个人资料、用户管理、API Key。

**普通用户状态**：

- 根据真实角色与权限显示摘要。
- 至少保留个人资料；用户管理仅在 `user:read` 时显示；API Key 仅在 `api:view` 时显示。
- 普通用户不能看到灰掉的管理员入口，应直接隐藏。

**响应式**：摘要桌面三列，手机单列；快捷入口桌面最多三列，手机单列。

### 9.4 个人资料 `Console / Profile`

**顶部摘要条**：登录账号、角色、账号状态。角色编码允许换行，账号状态显示 `已启用` 标签。

**主体**：

- 桌面两列，左侧 `基本信息`，右侧 `修改密码`，中间只用分隔线，不把每列做成嵌套卡片。
- 手机纵向排列，两区之间使用水平分隔线。

**基本信息字段**：姓名、邮箱、手机号、所属学院；按钮 `保存资料`。

**修改密码字段**：当前密码、新密码、确认新密码；新密码 6 至 20 位；按钮 `更新密码`。成功后清空所有密码字段。

两个表单的提交和加载状态相互独立。

### 9.5 用户管理 `Console / Users`

**页面级操作**：

- 次按钮 `下载模板`，下载 `.xlsx`。
- 次按钮 `批量导入`，只接受 `.xlsx`，最大 2MB。
- 主按钮 `新建用户`。
- 按权限隐藏对应按钮，不显示禁用占位。

**筛选**：用户名关键词输入、查询、重置。

**表格列**：

| 列 | 表现 |
| --- | --- |
| 用户名 | 主要文本，不显示无接口支持的排序控件 |
| 姓名 | 普通文本，空值显示 `-` |
| 学院 | 长文本截断 Tooltip |
| 角色 | 多个细边框 Tag，如 `ROLE_ADMIN`、`ROLE_STUDENT` |
| 状态 | 启用/停用 Switch |
| 操作 | 编辑、重置密码、删除图标按钮 |

**新建/编辑用户 Dialog**：

- 桌面两列表单，最大宽 620px；手机单列全屏或近全屏。
- 字段：用户名、初始密码、姓名、邮箱、手机号、所属学院、角色多选。
- 新建时显示用户名和初始密码，默认角色 `ROLE_STUDENT`。
- 编辑时用户名禁用且不显示密码字段。
- 至少选择一个角色。
- 底部：取消、保存。

**重置密码**：独立确认式输入 Dialog，标题 `重置密码`，明确显示目标用户名；新密码 6 至 20 位。

**删除用户**：危险确认框，显示用户名和 `该操作不可撤销`。

**批量导入结果 Dialog**：

- 顶部摘要：共读取 N 行、成功 N 行、失败 N 行。
- 无失败时绿色成功提示；有失败时黄色警告提示。
- 错误表列：行号、用户名、失败原因，最大高度 360px。
- 唯一操作 `完成`。

### 9.6 角色权限 `Console / Roles`

**页面级操作**：主按钮 `新建角色`，需要 `role:update`。

**表格列**：角色名称、角色编码、说明、权限数、操作。

**行操作**：分配权限、编辑角色、删除角色。使用设置、编辑、删除图标和 Tooltip。

**新建/编辑角色 Dialog**：

- 字段：角色名称、角色编码、说明。
- 角色编码必须以 `ROLE_` 开头，仅大写字母、数字和下划线。
- 说明使用 3 行 Textarea。
- 底部：取消、保存。

**分配权限 Drawer**：

- 标题 `分配权限 - {角色名称}`。
- 使用带复选框的层级树，默认展开全部节点。
- 节点显示权限名称，父子层级必须清楚。
- 底部固定：取消、保存权限。

**删除角色**：危险确认框，明确显示角色名称。

### 9.7 操作日志 `Console / Operation Logs`

**筛选**：操作人、模块、查询、重置。

**表格列**：操作人、模块、操作、说明、IP、耗时(ms)、状态、时间、详情。

- 成功状态为绿色 `成功`，失败状态为红色 `失败`。
- 说明长文本截断并提供 Tooltip。
- 详情列使用查看图标。

**日志详情 Drawer**：

- 标题 `日志详情`，右侧 520px。
- 单列键值布局：操作人、模块、操作、方法、参数摘要、IP、耗时、结果、错误、时间。
- 参数摘要和错误信息允许换行并使用等宽/技术文本样式，但不能暴露密码或 Token 示例。
- 无错误时显示 `-`。

### 9.8 API Key `Console / API Keys`

**页面级操作**：主按钮 `创建 API Key`。

**筛选**：应用名称、查询、重置。

**表格列**：应用名称、创建人 ID、每分钟上限、累计调用、过期时间、状态、操作。

- 无过期时间显示 `永不过期`。
- 状态使用启用/禁用 Switch。
- 操作仅为删除图标。

**创建 API Key Dialog**：

- 应用名称，最多 100 字符。
- 每分钟调用上限，数字步进器，范围 1 至 10000，默认 100。
- 过期时间，日期时间选择器；辅助文字 `留空表示永不过期`。
- 底部：取消、创建。

**一次性密钥 Dialog**：

- 创建成功后立即出现，标题 `API Key 已创建`。
- 红色/警告色说明：`完整密钥只显示一次，关闭后无法再次查看。`
- 密钥在浅灰代码区域完整换行显示，右侧是复制图标按钮。
- 禁止点击遮罩关闭，禁止右上角关闭，仅允许点击 `我已保存`。
- 关闭时清除界面上的密钥值。

**删除 API Key**：危险确认框，显示应用名称。

### 9.9 API 调用统计 `Console / API Calls`

**顶部统计条**：累计调用、成功调用、失败调用、平均耗时。四项同级、无外层嵌套卡片；平均耗时保留 1 位小数并显示 `ms`。

**筛选**：接口路径、查询、重置。

**表格列**：Key ID、方法、接口路径、参数名/摘要、IP、耗时(ms)、状态码、调用时间。

- 方法可使用克制的文本 Tag，例如 GET/POST，不为每种方法制造大色块。
- 状态码小于 400 使用绿色，大于等于 400 使用红色，同时保留数字。
- 接口路径使用等宽字体并允许 Tooltip。
- 不生成趋势图，因为当前没有时间序列统计接口。

**响应式**：统计条桌面四列、平板两列、手机单列或两列；表格水平滚动。

### 9.10 API 文档 `Console / API Docs`

- 页面头部标题 `API 文档`，右侧次按钮 `刷新`。
- 主体为内嵌 `/swagger-ui/index.html` 的 iframe，不重新设计 Swagger 内部内容。
- iframe 使用完整可用宽度，高度为视口减去顶部栏和页面头，最小 560px。
- 外框仅 1px `border`，不得把 iframe 放入多层卡片。
- 加载时显示轻量 Skeleton/Spinner；加载失败显示明确错误和重试按钮。

## 10. 全局状态规范

每个异步页面至少设计以下状态：

1. 默认有数据状态。
2. 首次加载状态：标题和筛选栏保留，数据区 Skeleton。
3. 空状态：简洁图标、`暂无...` 文案，若用户有创建权限则提供对应主操作。
4. 筛选无结果：显示 `未找到匹配结果` 和 `清除筛选`，不要误写成系统无数据。
5. 请求失败：页面内错误区或 Toast，保留重试入口。
6. 保存中：提交按钮加载，表单不可重复提交。
7. 保存成功：短 Toast，不跳到无关页面。
8. 字段错误：字段级错误，不只显示全局 Toast。
9. 无权限：菜单隐藏；路由守卫返回工作台，不生成独立 403 页面。
10. Token 失效：返回登录页，不设计新的会话恢复页面。

## 11. 示例内容规范

- 使用真实感中文姓名和学院，例如 `李明`、`张老师`、`计算机学院`，不要使用 `Lorem ipsum`。
- 角色必须使用真实编码：`ROLE_ADMIN`、`ROLE_STUDENT`、`ROLE_TEACHER`、`ROLE_ANALYST`、`ROLE_COLLEGE_ADMIN`。
- API 路径可使用 `/api/open/v1/platform/status` 等真实风格示例。
- 时间统一示例为 `2026-07-10 16:30:25`。
- 空字段统一显示 `-`，过期时间例外显示 `永不过期`。
- 所有可见文案使用简体中文；保留 API、Key、IP、ID、Token、Swagger、角色编码和 HTTP 方法等技术术语。

## 12. 可访问性与可用性

- 所有文字、图标和状态颜色达到 WCAG 2.1 AA。
- 所有交互元素支持键盘 Tab，焦点清晰可见。
- 图标按钮必须有 Tooltip、`aria-label` 或等价可访问名称。
- 表单错误与字段通过语义关联，不能只改变颜色。
- Switch 必须有邻近状态文字或无障碍状态描述。
- 弹窗打开后焦点进入弹窗，关闭后回到触发按钮。
- 手机触控目标最小 44 x 44px。
- 任何视口下，文字不能与按钮、表格、标题或后续内容重叠。

## 13. 动效规范

- Hover/选中/展开过渡 120ms 至 180ms，`ease-out`。
- 侧栏宽度变化 180ms，内容布局不能跳动。
- Dialog/Drawer 使用短淡入和位移动效。
- 尊重 `prefers-reduced-motion`，关闭非必要动效。
- 不使用数字滚动、卡片漂浮、背景运动或循环动画。

## 14. Stitch 负面约束

生成时必须避免以下常见错误：

- 不要做成营销首页或数据大屏。
- 不要使用大面积深绿、深蓝、紫色或黑色背景。
- 不要使用渐变、玻璃拟态、发光、光球、波纹背景或装饰性网格。
- 不要生成巨大欢迎语、超大 KPI、圆角 16px 以上的卡片或胶囊按钮。
- 不要将每个页面区块都做成卡片，不要卡片套卡片。
- 不要新增通知、搜索、聊天、导出、排序、图表或批量操作，除非本文件明确要求。
- 不要把学生、教师设计成独立产品或独立登录入口。
- 不要把被权限隐藏的功能显示成禁用菜单。
- 不要修改路由名称、字段名称、权限编码和角色编码。
- 不要在 UI 中展示默认密码、真实 Token 或完整示例密钥。
- 不要使用自定义手绘 SVG 图标。
