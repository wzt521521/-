## 4. 代码链路审计

### 4.1 登录与认证

```text
LoginView -> auth API -> Axios -> AuthController -> AuthService
-> AuthenticationManager -> SysUser/Role/Permission -> BCrypt
-> JwtTokenProvider -> TokenStore -> TokenResponse -> Pinia/LocalStorage
```

每次受保护请求都会重新从数据库加载用户状态和当前权限。因此用户被禁用后，旧 Access Token 也会在下一次请求立即被拒绝，不长期信任 Token 内的旧权限。

### 4.2 Token 刷新

```text
并发 401 -> createRefreshCoordinator（只发一个请求）
-> /api/auth/refresh -> JWT 类型/签名/过期校验
-> Redis Lua 比较并删除 Refresh Token
-> 签发新双 Token -> 重放等待中的原请求
```

Redis 通过单条 Lua 脚本原子消费 Refresh Token；测试环境通过同步移除实现相同语义。并发测试证明 32 个消费者中只有 1 个成功。

### 4.3 RBAC 与数据范围

```text
前端菜单/路由/按钮权限（体验层）
-> JWT Filter（认证层）
-> @PreAuthorize（功能授权层）
-> DataScopeProvider（数据授权层）
-> Service/Repository 查询条件
```

后端是最终授权边界。学院管理员和教师必须按学院过滤，学生必须按用户 ID 过滤；前端隐藏不能替代后端限制。

### 4.4 开放 API

```text
X-API-Key -> SHA-256 查询、状态和过期校验
-> Redis 每分钟限流 -> JWT Filter -> 业务 Controller
-> ApiCallLog + totalCalls -> 统一响应与限流头
```

Swagger 中开放接口的一个 Security Requirement 同时包含 `bearerAuth` 和 `apiKeyAuth`，文档语义与过滤链一致。

### 4.5 Excel 导入

```text
UserListView -> multipart/form-data -> UserManagementController
-> Apache POI -> 表头/行数/DTO 校验 -> UserManagementService.create
-> 每行独立提交 -> 汇总成功数和行级错误
```

密码不进入返回结果或操作日志，错误仅包含行号、用户名和原因。
