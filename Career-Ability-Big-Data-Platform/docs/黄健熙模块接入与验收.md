# 黄健熙模块接入与验收

## 1. 已交付能力

- Spring Boot 单模块集成工程中的认证、开放 API、统一响应、分页、错误码与全局异常能力。
- BCrypt 密码、JWT 双 Token、Refresh Token 原子轮换、退出黑名单。
- Redis 登录失败锁定：5 次失败后锁定 15 分钟。
- 五角色 RBAC、方法级权限、前端路由/菜单/按钮权限。
- `ALL`、`COLLEGE`、`SELF` 三档后端数据范围契约。
- 用户、角色、权限、密码重置、Excel 批量导入和操作日志管理。
- Vue 3 登录注册、主布局、个人资料和系统管理页面。
- API Key 哈希存储、JWT + API Key 双认证、Redis 分钟限流、调用日志与统计。
- OpenAPI 安全方案、在线 Swagger 和 Postman 联调集合。

## 2. 业务模块接入契约

业务接口默认需要 JWT。Controller 使用权限编码限制功能，Service 或 Repository 使用数据范围限制记录集合，不能只依赖前端隐藏。

```java
@PreAuthorize("hasAuthority('position:view')")
@GetMapping("/api/positions")
public ApiResponse<?> list() {
    return ApiResponse.success(positionService.list());
}
```

当前用户与数据范围通过公共接口注入：

```java
private final CurrentUserProvider currentUserProvider;
private final DataScopeProvider dataScopeProvider;

CurrentUser user = currentUserProvider.requireCurrentUser();
DataScope scope = dataScopeProvider.requireDataScope();
```

数据范围映射：

| 角色 | 范围 | 查询约束 |
| --- | --- | --- |
| `ROLE_ADMIN` | `ALL` | 可查询全校数据 |
| `ROLE_ANALYST` | `ALL` | 可查询全校数据 |
| `ROLE_COLLEGE_ADMIN` | `COLLEGE` | 必须按 `scope.college` 过滤 |
| `ROLE_TEACHER` | `COLLEGE` | 必须按 `scope.college` 过滤 |
| `ROLE_STUDENT` | `SELF` | 必须按 `scope.userId` 过滤 |

学院角色未配置学院时会返回 403。未知自定义角色按最小权限降级为 `SELF`。

关键写操作使用公共 `@Log` 注解。切面仅持久化参数类型，不持久化请求对象内容，因此密码、Token 和 API Key 原文不会进入审计参数。

## 3. 认证约定

| 项目 | 约定 |
| --- | --- |
| Access Token | 2 小时，`Authorization: Bearer <token>` |
| Refresh Token | 7 天，成功刷新后原 Token 立即失效 |
| 自助注册 | 仅 `ROLE_STUDENT`、`ROLE_TEACHER` |
| 高权限角色 | 仅管理员分配 |
| 未认证 | HTTP 401，响应体 `code=401` |
| 权限不足 | HTTP 403，响应体 `code=403` |
| 登录锁定 | 5 次失败，15 分钟，HTTP 429 |

前端认证状态只能通过 `src/utils/auth.js` 和 Pinia User Store 修改。并发 401 共用一个刷新请求，刷新失败清空三项 LocalStorage 并跳转登录页。

## 4. 开放 API 约定

`/api/open/v1/**` 必须同时携带：

```http
Authorization: Bearer <accessToken>
X-API-Key: <apiKey>
```

API Key 原文只在创建响应中显示一次，数据库仅保存 SHA-256。每个 Key 使用独立的每分钟限额，响应返回 `X-RateLimit-Limit` 和 `X-RateLimit-Remaining`。

目前可执行的开放链路是：

| 方法 | 路径 | 状态 |
| --- | --- | --- |
| GET | `/api/open/v1/platform/status` | 已实现，用于验证双认证、限流和调用审计 |

以下是已经冻结的预测和业务开放接口契约。B 负责认证、限流、日志和文档接入，具体计算与响应数据由对应业务模块实现：

| 方法 | 预测接口 | 作用 | 业务负责人 |
| --- | --- | --- | --- |
| GET | `/api/open/v1/forecasts/job-demand` | 返回岗位或行业需求预测、模型版本和质量指标 | C |
| GET | `/api/open/v1/forecasts/salary-trend` | 返回地区、岗位或行业的薪资趋势预测 | C |
| GET | `/api/open/v1/forecasts/skill-demand` | 返回技能需求趋势、置信区间和数据时间范围 | C |
| GET | `/api/open/v1/positions` | 分页查询可公开的岗位数据 | C |
| GET | `/api/open/v1/skills/hot` | 返回热门技能及需求频率 | C |
| GET | `/api/open/v1/cities/ranking` | 返回城市岗位量和薪资排名 | C |
| GET | `/api/open/v1/reports` | 查询调用方授权范围内的报告 | D |
| GET | `/api/open/v1/recommendations` | 返回授权用户或画像参数的岗位推荐 | D |

预测接口必须返回模型版本、训练数据截止时间、样本量、适用范围和质量指标；历史数据不足时统一返回 422，不能伪造预测结果。

## 5. 用户 Excel 导入

管理员在用户管理页下载模板并上传 `.xlsx`：

- 必填列：`username`、`password`、`roleCodes`。
- 可选列：`realName`、`email`、`phone`、`college`。
- 多角色使用逗号或分号分隔。
- 单文件最多 2 MB、1000 条数据。
- 有效行独立提交，非法行返回 Excel 行号、用户名和失败原因。
- 返回结果不包含密码，审计日志不读取文件内容。

## 6. 验证覆盖

后端自动化验证覆盖：

- 注册、登录、当前用户、刷新轮换、重复刷新和退出黑名单。
- JWT 正常、过期、篡改和 Token 类型混用。
- 无 Token 401、权限不足 403、禁用用户立即失效。
- 登录失败锁定和 Refresh Token 并发原子消费。
- 管理员用户查询和 Excel 部分成功导入。
- API Key 哈希、缺失、非法、禁用、过期、限流和双认证。
- 操作日志敏感字段保护和 OpenAPI 双安全方案。

前端自动化验证覆盖认证存储、权限判断和并发刷新单航班逻辑，并执行 ESLint 与生产构建。

## 7. 当前外部依赖与限制

- 岗位/分析模块已在当前工程中，仍需在集成分支验证其认证和数据范围；画像/推荐模块尚未完整接入，不能把对应业务接口标记为已联调。
- 当前机器未安装 Docker，Compose 只能做静态检查，无法在本机完成 MySQL + Redis + Nginx 容器实跑。
- `data-pipeline/Dockerfile` 和五服务 Compose 已存在，但仍需在安装 Docker 的 Linux 或 CI 环境执行完整容器验收。
- 为保持项目要求的 Node 18 基线，Vite 5 的两项开发服务器审计风险暂未跨大版本升级。生产依赖审计为 0，开发服务器默认仅监听 `127.0.0.1`。
