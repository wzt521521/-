# v1.0.0 发行范围与版本基线

> 基线提交：`2e2040c3cfc48829b9f7adcf224a4c54f7b39d84`

>
> 冻结日期：2026-07-14

## 1. 决策摘要

- `v1.0.0` 的报告交付格式为 PDF；Word 和 Excel 明确进入 `v1.1`，不阻塞 `v1.0.0`。
- `v1.0.0` 以当前单体 Spring Boot 后端、Vue 前端、Python ETL、MySQL 和 Redis 五服务架构为准，不在发行稳定期重构 Maven 多模块或引入新的大数据平台组件。
- P0 和 P1 任务全部关闭、RC 全链路验收通过后才能发布正式版；P2 可经风险评审移入 `v1.1`。
- 本文件消除需求大纲“三格式报告”和项目整体规划“PDF 优先、其他格式后续增强”之间的冲突，并作为本次发行解释基线。

## 2. v1.0.0 必须交付

1. 五服务可从干净环境构建、启动、健康检查、持久化、升级和回滚。
2. JWT 登录、刷新轮换、RBAC、用户/角色管理、操作审计和数据范围形成可验证闭环。
3. 采集源、采集任务、执行日志、调度、启停、重试和失败恢复形成真实运行链路。
4. CSV 导入、Redis 队列、ETL 清洗、MySQL 入库具备幂等、重试和坏消息处理能力。
5. 岗位查询、七维统计、可视化大屏、定时统计和缓存降级在真实 MySQL/Redis 上通过验收。
6. 用户画像、五维推荐、技能差距、TOP20 和缓存失效行为通过授权与业务测试。
7. PDF 报告的生成、筛选、状态流转、下载、删除、持久化、中文字体和安全转义通过验收。
8. 开放 API 提供至少一条受 API Key、签名/限流、审计和数据范围约束的业务数据链路。
9. CI、密钥扫描、依赖/许可证审查、监控、备份恢复、文档和发行产物达到任务清单定义。

## 3. 明确不属于 v1.0.0

- Word、Excel 报告导出和报告图表增强进入 `v1.1`。
- Kafka、Spark、Hive、HDFS、Scrapy、WebSocket 和告警中心等规划 17-23 项保持后期选做。
- Spring Boot 大版本升级、微服务拆分和 Maven 多模块重构不在本次发行稳定期实施。
- 新需求只有在修复 P0/P1 所必需或经四人共同确认后才能进入 `v1.0.0`。

## 4. 范围变更流程

1. 变更提议必须使用任务清单 ID 创建 GitHub Issue，并说明进入 `v1.0.0` 的必要性、风险和替代方案。
2. P0/P1 范围变化须由四名成员在 Issue 中明确确认；未确认前保持当前冻结范围。
3. 批准后同步更新本文件、发行任务清单和对应 Issue，不接受只在聊天中口头变更。
4. 每个 PR 只关联一个可验证任务 ID，并合入 `release/1.0.0-integration`。
